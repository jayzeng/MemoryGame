# Squishmallow Memory Parade – Product Specification

## 1. Product Overview

**Product Name:** Squishmallow Memory Parade  
**Genre:** Casual Memory / Educational  
**Target Age:** 5–10  
**Primary Platforms:** iPad, Tablet, Mobile (Portrait-first)  
**Session Length:** 2–5 minutes  
**Monetization:** None / Parent-approved future expansion only  

**Product Vision:**  
Create a joyful, stress-free memory game that builds cognitive skills through emotional attachment, playful feedback, and adaptive difficulty—designed to feel like a hug, not a test.

---

## 2. Goals & Success Metrics

### Player Goals
- Have fun matching Squishmallows
- Feel successful regardless of skill level
- Build attachment to characters

### Parent Goals
- Educational value (memory, focus)
- No pressure, no punishment
- Safe for independent play

### Success Metrics
- Average session length: 3–6 minutes
- Return rate (day 7)
- Level completion without rage quits
- Parade Book collection engagement

---

## 3. Target Audience Breakdown

### Ages 5–6
- Small boards (2×3, 2×4)
- Identical matching
- Strong hints & assistance

### Ages 7–8
- Medium boards (3×4, 4×4)
- Faster pacing
- Light accessory matching

### Ages 9–10
- Larger boards (4×5+)
- Character + attribute matching
- Optional challenge mode (no timers by default)

---

## 4. Core Gameplay Loop

1. Choose Parade World
2. Select next level
3. Flip cards to find matches
4. Receive immediate positive feedback
5. Collect Squishmallows & stickers
6. Unlock next level or world
7. Repeat

---

## 5. Game Mechanics

### Memory Matching
- Flip two cards per turn
- Matched cards remain revealed
- Unmatched cards flip back gently

### Match Types
- Character + Character
- Character + Accessory
- Character + Emotion (advanced)

---

## 6. Adaptive Difficulty System

The game dynamically adjusts based on:
- Time between flips
- Repeated mismatches
- Level completion speed

### Adjustments Include
- Board size reduction
- Slower animations
- Automatic hints
- Increased visual cues

**Design Rule:** The game helps silently; the child never feels corrected.

---

## 7. Power-Ups & Assistance

### Soft Power-Ups (Auto-triggered)
- Peek-a-Boo: Brief reveal
- Squish Paw: Wiggle hint
- Sleepy Mode: Slower pace

No consumables. No scarcity.

---

## 8. Progression & Worlds

### Worlds
1. Cozy Bedroom
2. Candy Carnival
3. Forest Friends
4. Ocean Splash
5. Space Squish

Each world contains:
- 8–12 short levels
- Unique Squishmallows
- Themed music & visuals
- Parade Finale level

---

## 9. Reward System

### Rewards
- Stars (non-competitive)
- Stickers
- Squishmallow unlocks
- Accessories (hats, bows)

### Design Principles
- No failure screens
- Rewards for effort
- Celebrations are short and skippable

---

## 10. Parade Book (Collection System)

- Visual grid of unlocked Squishmallows
- Tap to view personality, favorites
- Customize with accessories
- Locked Squishmallows shown as silhouettes

Purpose: emotional bonding & long-term motivation

---

## 11. Audio & Feedback Design

- Gentle background music
- Soft squish sounds on flip
- Encouraging voice lines:
  - “Nice try!”
  - “You remembered!”
  - “Almost!”

No negative sounds or buzzers.

---

## 12. Accessibility & Safety

### Accessibility
- No timers by default
- Colorblind-friendly palettes
- Large touch targets
- Optional reduced motion

### Safety
- Offline play
- Parent Gate for settings
- No ads
- No external links

---

## 13. Multiplayer / Family Mode (Optional)

- Pass-and-play turn taking
- Parent + child cooperative mode
- Shared celebration screens

---

## 14. Technical Considerations (High Level)

- Deterministic shuffle with seed
- Board size defined by config
- Difficulty engine runs locally
- Asset bundles per world

---

## 15. Future Expansion Ideas

- Seasonal Squishmallows
- Holiday parade worlds
- Daily Memory Moment
- Classroom mode
- Accessibility-first variants

---

## 16. Product Principles (Non-Negotiable)

- No punishment
- No urgency
- No comparison
- Delight over difficulty
- Emotional safety first

---

## Appendix: What This Game Is NOT

- Not competitive
- Not timed by default
- Not monetization-driven
- Not text-heavy

End of Product Specification.
